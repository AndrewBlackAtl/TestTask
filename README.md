Тестовое задание

Необходимо сделать небольшой прототип аркадной игры с видом сверху.

### Что требуется?

- Игровой персонаж
    - Имеет здоровье
    - Может перемещаться с помощью **WASD**
        - Может проходить сквозь препятствия
    - Из персонажа раз в секунду в случайном направлении вылетает снаряд, который движется по прямой
        - При попадании в противника, наносит ему 25 единиц урона
    - При нуле здоровья:
        - Возможность перемещать персонажа блокируется
        - Стрельба снарядами прекращается
- 3D-камера, вид сверху, следует за персонажем игрока
- Противники
    - Имеют 100 единиц здоровья
    - Раз в 7 секунд за пределами видимости камеры появляются противники
    - Двигаются в сторону игрока
        - Останавливаются, когда подходят близко к игроку
    - Наносят игроку 5 единиц урона в секунду, когда находятся рядом с ним
    - При нуле здоровья - умирают
- Монетки
    - При смерти противника с шансом 50% выпадает «Монетка»
    - Когда игрок находится рядом с монеткой, она исчезает, а игроку добавляется +1 к значению ресурса
- UI
    - В левом верхнем углу экрана, выводится цифра текущего здоровья игрока
    - В правом верхнем углу экрана выводится значение ресурса игрока (Кол-во собранных монеток)
    - Над противниками должны выводиться цифры здоровья (С помощью **Unity UI**)
    - После смерти персонажа открывается экран проигрыша с кнопкой перезапуска

### Особенности выполнения

- Не нужны графика, анимации, эффекты и звуки
    - Рекомендуется делать на 3D-примитивах
- Если необходимо, для упрощения работы можно использовать бесплатные плагины и библиотеки
- Все числовые параметры из тестового задания (здоровье, урон, скорость движения персонажей, шанс спавна монетки из врагов) - должны иметь возможность понятным образом настраиваться из редактора **Unity**
- Если есть опыт работы с **ECS** фреймворками ****и/или **Data Oriented** подходом, то следует выполнить задание с их использованием (Иначе их использовать не стоит)
    - Желательно не **DOTS**

### Время на выполнение

- Нет каких-либо сроков, можно работать в комфортном темпе.
- Рекомендуется не тратить много времени на отдельные элементы задания, а сконцентрироваться на выполнении его целиком.

### В каком виде сдавать тестовое

- Загрузить файлы проекта на **GitHub**, или сделать **zip**-архив и передать нам.
- Необходимо включить только обязательные папки **Unity**-проекта (Например, папка **Library** не должна быть включена).
